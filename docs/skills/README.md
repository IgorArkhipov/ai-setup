# Document Skills

This directory separates document generation guidance from document review prompts.

Use these docs when working with feature documentation in this repository:

1. `brief` captures the problem and desired outcome.
2. `spec` defines the solution, boundaries, invariants, and acceptance criteria.
3. `plan` explains the codebase-specific implementation order and grounding.

## Generation

- `brief-generation.md`
- `spec-generation.md`
- `plan-generation.md`

## Review

- `brief-review.md`
- `spec-review.md`
- `plan-review.md`

## Shared References

- `spec-template.md` is the default output shape for generated specs.

## Usage Notes

- Use generation docs when drafting a new document from source material.
- Use review docs when checking whether an existing document is ready for the next stage.
- Prefer linking documents together explicitly, for example brief -> spec -> plan.

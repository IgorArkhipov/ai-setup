# Prompt: Spec Improve Loop

Work according to `homeworks/hw-4/full/process-specs/spec-improve-loop.md`.

Inputs:

- improved brief path;
- draft spec path;
- target improved spec path;
- runner state directory.

Rules:

1. Read the improved brief and draft spec before writing.
2. Improve the spec in English.
3. Preserve scope from the brief.
4. Make requirements, non-scope, constraints, acceptance scenarios, verification, risks, and open questions explicit.
5. Keep the spec actionable enough for an implementation plan.
6. Return a runner result with status `done`, `blocked`, or `escalation`.

Expected output sections:

- improved spec;
- checks performed;
- changes made;
- open questions;
- status;
- next action.

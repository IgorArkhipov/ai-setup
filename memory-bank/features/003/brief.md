# Feature 003: First End-To-End Provider Validation With Claude

## What problem are we solving?

AgentScope's shared discovery and mutation concepts are still unproven until they work against a real provider that users already rely on. Without one end-to-end provider integration, the project remains mostly infrastructure and cannot yet demonstrate that its model delivers real value in day-to-day workflows.

## Who is this for?

- Claude Code users who want visibility into and control over the supported parts of their local setup
- The AgentScope maintainer, who needs a real provider baseline before extending the product to other providers

## Where did this task come from?

In the feature-based memory bank, it turns the next milestone into a concrete user-facing problem: proving the product on one real provider before broadening support further.

## What outcome do we want?

Claude users should be able to inspect supported Claude configuration through AgentScope and manage the supported parts of that setup without falling back to manual configuration edits. The project should gain a validated reference integration that can guide future provider work.

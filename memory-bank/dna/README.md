# DNA Index

DNA is the constitution of the project documentation. It defines principles, documentation rules, the frontmatter schema, and lifecycle rules.

- [Principles](principles.md) - the foundational rules of the project: SSoT, atomicity, and progressive disclosure. Read this first.
- [Document Governance](governance.md) - SSoT implementation and the dependency tree. Answers the question: who owns a fact.
- [Frontmatter Schema](frontmatter.md) - the schema for frontmatter fields.
- [Document Lifecycle](lifecycle.md) - maintenance rules and the sync checklist.
- [Cross-references](cross-references.md) - rules for two-way navigation between code and docs.

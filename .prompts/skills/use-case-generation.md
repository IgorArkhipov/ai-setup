# Use-Case Generation

Use this skill when the work introduces or changes a stable project-level scenario.

## Execution

1. Use `../memory-bank-create-use-case.md`.
2. Immediately follow with `use-case-review.md`.
3. If review finds issues or the use case changes materially, use `review-loop.md`.

## Preserve

- stable actor, trigger, preconditions, main flow, and postconditions;
- project-level scenario ownership rather than feature-local acceptance only;
- no implementation sequencing or feature-level verification logic.

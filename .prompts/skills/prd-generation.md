# PRD Generation

Use this skill when the work is initiative-level and needs product framing before decomposition into features.

## Execution

1. Use `../memory-bank-create-prd.md`.
2. Immediately follow with `prd-review.md`.
3. If review finds issues or the PRD changes materially, use `review-loop.md`.

## Preserve

- initiative scope, not single-feature scope;
- explicit problem, users, goals, non-goals, and success metrics;
- no implementation sequencing or architecture ownership inside the PRD.

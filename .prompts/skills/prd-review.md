# PRD Review

Use this skill to review a governed PRD before downstream feature work depends on it.

## Execution

1. Use `../memory-bank-review-prd.md`.
2. If any criterion fails, fix the PRD and review again through `review-loop.md`.
3. If independent confirmation is needed, trigger Claude Code MCP through `review-loop.md`.

## Preserve

- initiative-vs-feature boundary;
- explicit users and business value;
- measurable outcomes;
- no hidden implementation design.

# Implementation-Plan Generation

Use this skill when a sibling `feature.md` is already design-ready and execution sequencing needs a governed owner.

## Execution

1. Use `../memory-bank-create-implementation-plan.md`.
2. If the Design Ready gate fails, stop and repair the feature document first.
3. Immediately follow with `implementation-plan-review.md`.
4. If review finds issues or the plan changes materially, use `review-loop.md`.

## Preserve

- derived-document boundary;
- grounded current-state context;
- atomic dependency-ordered steps;
- explicit test strategy and approval gates.
